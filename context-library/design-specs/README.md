# Design Specs (Reference)

This folder stores completed and shipped UX specs. The copilot uses them as reference when drafting new specs: for patterns, language, and how your team structures work.

---

## What Belongs Here

Finalized specs that have been:
- Signed off by PM and/or engineering
- Shipped or currently in development

**Active drafts go in `outputs/design-specs/`.** Move a spec here once it's finalized.

---

## File Naming

`[feature-name-kebab-case]-[date].md`

Examples:
- `onboarding-step-2-email-2025-03.md`
- `filter-bar-v2-2025-06.md`
- `empty-states-overhaul-2025-01.md`

---

## Learning Value

When generating new specs, the copilot may reference these to:
- Match your team's level of detail and format
- Avoid re-solving problems already solved
- Check if edge cases from a prior spec apply to the new one
