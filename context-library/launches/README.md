# Launches

Major UI release notes, go-live summaries, and design post-mortems.

Optional but useful for tracking what shipped and capturing what you learned.

---

## What Belongs Here

- **Release notes** (design perspective): what changed visually or functionally for users
- **Launch post-mortems**: what went well, what to do differently
- **Before/after summaries**: screenshot or Figma link comparisons, key metrics if available

---

## File Naming

`[feature-or-release]-launch-[date].md`
