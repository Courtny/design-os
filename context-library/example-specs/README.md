# Example Specs

Reference specs used to calibrate the copilot's output format and quality bar.

Add 1–3 examples of specs that represent the level of detail, voice, and structure your team expects. These don't need to be your most complex work — just clear examples of a well-written spec.

---

## What Belongs Here

- One lightweight spec (simple component change or small feature)
- One medium-complexity spec (new feature with multiple states and edge cases)
- One handoff-ready spec (shows your full annotation style and Storybook component references)

The copilot uses these as models when you run `/design-spec-draft`.
