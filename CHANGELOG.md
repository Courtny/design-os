## [1.0.0] — 2026-03-31

### Initial Release

**Skills (13):**
- `/connect-mcps` — Connect Figma and other tool MCPs
- `/ux-brief` — Create design briefs from problem statements or PRDs
- `/critique-prep` — Build critique agendas with structured feedback prompts
- `/design-spec-draft` — Draft UX specs with flows, states, and edge cases
- `/prototype` — Generate HTML/CSS/JS prototypes using design system components
- `/research-synthesis` — Turn session notes and transcripts into structured insights
- `/accessibility-review` — WCAG 2.1 AA and heuristic review
- `/meeting-notes` — Process critique and sync notes into decisions and action items
- `/visual-critique` — Critique a design from an attached image
- `/figma-critique` — Critique a Figma file with optional comment posting
- `/color-review` — Semantic color and token adherence review
- `/content-review` — Microcopy and content hierarchy review
- `/handoff-check` — Pre-handoff spec completeness check

**Sub-agents (5):**
- PM alignment, engineering feasibility, content design, accessibility, brand

**Context library:**
- Product context template with filled example
- 4 writing style files (internal, executive, engineering handoff, user-facing)
- Stakeholder profile template
- Design system reference (Storybook, Figma, principles, component UX)
- 37 foundation files (HCI laws, cognitive effects, Gestalt, prioritization frameworks)
- 3 example specs at different complexity levels
- 2 methodology docs (critique norms, handoff checklist)

**Templates (8):**
- UX brief, design spec, critique agenda, component UX, accessibility checklist, meeting notes, research synthesis, decision log

**Setup:**
- First session checklist (Claude Code)
